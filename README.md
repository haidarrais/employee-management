## Overview

The system allows employees to mark attendance by scanning a time-limited QR code generated by an admin or manager. Attendance is validated against a geofence radius around the workplace. A configurable schedule engine calculates late minutes, overtime, and daily pay per employee — including special rates for public holidays and authorised Sunday work.

```bash

# 2. Install PHP dependencies
composer install

# 3. Install Node dependencies and build assets
npm install

cp .env.example .env
```

The encryption key for QR tokens is derived from `APP_KEY`. No separate key is required.

## Features

### Authentication & MFA

- Session-based login for the web interface
- Token-based login (Sanctum) for the mobile API
- Optional TOTP two-factor authentication via Google Authenticator
- MFA can be enabled/disabled per user from the Profile page
- Session timeout warning modal for mobile sessions

### QR Code Attendance

1. An admin or manager navigates to **Generate QR Code**
2. A single-use, AES-256-GCM encrypted token is generated and stored with a 5-minute expiry
3. The QR image is displayed with a live countdown timer (green → yellow → red)
4. An employee scans the code using the mobile app camera
5. The API validates: token hash match → not expired → not already used → geofence check
6. On success, an `AttendanceRecord` is created with status `confirmed`

**QR token security:**
- Raw token is never stored — only a SHA-256 hash
- Encrypted token (IV + GCM tag + ciphertext, base64-encoded) is embedded in the QR image
- Single-use: marked `used` immediately on first valid scan

### Working Hours Schedule

Configured at `/reports/schedule`. Stored in `system_configs` under key `working_hours`.

- Per-day-of-week segments (up to two per day: morning + afternoon)
- Each day can be marked as **Off**
- Default schedule:
  - Mon–Fri: 07:00–12:00, break, 13:00–16:00
  - Saturday: 06:30–12:00, break, 12:30–14:30
  - Sunday: Off
- Grace period (default 15 min) before late is counted
- Global daily base fee and overtime rate (overridden per employee)

### Public Holidays & Special Days

Managed at `/reports/schedule#special-days`.

Two types:
- **Public Holiday** — employee receives no base pay; all hours worked are paid at the holiday overtime rate
- **Overtime Sunday** — same pay logic; used for authorised work on Sundays or other off-days

Each special day can carry a **per-day rate override** that takes priority over the global holiday rate.

**Rate priority (highest to lowest):**
1. Per-day `rate_per_hour` in the special day entry
2. Global `holiday_overtime_rate_per_hour`
3. Employee's personal `overtime_rate`
4. Global `overtime_rate_per_hour`

### Payroll Report

Available at `/reports/payroll` for admin and management.

**Filters:** month picker + employee selector

**Summary table** (per employee):
- Days attended
- Total late minutes
- Total overtime
- Total fee (always recalculated live from current rates)

**Daily detail table** (per employee):
- Date, day badge (★ for holidays/special days)
- Check-in time (colour-coded: green = on time, orange = within grace, red = late)
- Check-out time
- Late minutes, overtime, daily fee
- **Timeline bar** — visual 06:00–18:00 window showing check-in dot, check-out dot, work segment shading, official start line, and grace end line

Fees are always recalculated live — changing an employee's rate is immediately reflected in the report without re-seeding.

### User Management

Available at `/admin/users` for admins.

- Search by name or email, filter by role
- Edit user: name, email, role, password, **daily rate**, **overtime rate**
- Per-employee rates override the global schedule defaults
- Leaving rate fields blank reverts to global defaults
- Delete user (cannot delete own account)

### Audit Logs

Available at `/audit-logs` for admins.

Logs every significant action:
- `qr_code_generated`, `qr_validate_success`, `qr_validate_failed`
- `qr_validate_not_found`, `qr_validate_geofence_failed`
- `qr_generate_access_denied`, `role_access_denied`
- `role_changed`, `login_success`, `login_failed`, `logout`

Each entry records: user, action, IP address, timestamp, and a metadata payload (expandable in the UI).

Audit logs are **immutable** — the model blocks updates and deletes at the Eloquent level.

## Payroll Calculation Logic

### Regular Working Day

```
scheduled_minutes = sum of all work segments for that day-of-week

late_minutes = max(0, check_in_time - (official_start + grace_period))

overtime_minutes = max(0, check_out_time - official_end)

late_deduction = base_fee × (late_minutes / scheduled_minutes)

overtime_pay = overtime_rate_per_hour × (overtime_minutes / 60)

daily_fee = max(0, base_fee - late_deduction) + overtime_pay
```

### Public Holiday / Special Overtime Day

```
# No base pay, no late deduction
worked_minutes = check_out_time - check_in_time

daily_fee = holiday_rate_per_hour × (worked_minutes / 60)
```

### Rate Priority

For `base_fee`:
1. Employee's `daily_rate` (if set)
2. Global `daily_base_fee` from schedule config

For `overtime_rate`:
1. Per-day `rate_per_hour` in the special day entry
2. Global `holiday_overtime_rate_per_hour` (on holidays/special days)
3. Employee's `overtime_rate` (if set)
4. Global `overtime_rate_per_hour`

## Services

### `WorkingHoursService`

Central calculation engine. Loaded from `system_configs.working_hours`.

```php
$whs = app(WorkingHoursService::class);

// Check if a date is a working day
$whs->isWorkingDay(Carbon::parse('2026-05-26'));  // true (Monday)
$whs->isWorkingDay(Carbon::parse('2026-05-31'));  // false (Sunday)

// Get official start/end for a date
$whs->officialStart(Carbon::parse('2026-05-30'));  // 06:30 (Saturday)
$whs->officialEnd(Carbon::parse('2026-05-26'));    // 16:00 (Monday)

// Check holiday / special day
$whs->isHoliday(Carbon::parse('2026-08-17'));           // true
$whs->isSpecialOvertimeDay(Carbon::parse('2026-08-17')); // true

// Compute everything at once
$result = $whs->compute($checkIn, $checkOut, $user);
// Returns: late_minutes, overtime_minutes, daily_fee, is_special_day, special_day_name
```

### `QRCodeService`

```php
$svc = app(QRCodeService::class);

$qrCode = $svc->generate($userId);          // creates DB record
$image  = $svc->generateImage($qrCode);     // returns data:image/png;base64,...
$result = $svc->validate($qrCode, $token);  // returns ['valid' => bool, 'message' => string]
```

### `EncryptionService`

AES-256-GCM encryption for QR tokens. Key derived from `APP_KEY`.

```php
$enc = app(EncryptionService::class);
$encrypted = $enc->encryptQRToken($rawToken);
$decrypted = $enc->decryptQRToken($encrypted);
$enc->isAvailable(); // round-trip test
```

### `AuditService`

```php
app(AuditService::class)->log('qr_code_generated', $userId, [
    'qr_code_id' => 42,
    'ip_address' => $request->ip(),
]);
```
